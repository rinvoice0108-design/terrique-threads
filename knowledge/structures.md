# Threads 포스트 구조 템플릿

숏폼 원고에서 검증된 구조를 Threads 텍스트 형식으로 변환.
포스트 유형별로 아래 구조를 기반으로 작성할 것.

---

## 핵심 원칙
- 훅(첫 줄): 스크롤을 멈추는 한 문장 — 본문 없이도 독립적으로 의미 있어야 함
- 단락: 2~3줄 이하, 빈 줄로 호흡
- 수치 필수: GSM, 온도(℃), %, 소재명 등 근거 있는 정보만
- AI 나열식 금지: "다양한", "효과적인" 대신 구체적 표현
- 마무리: 댓글 유도 or 부드러운 권유 (강매 절대 금지)

---

## 구조 1. 통념파괴형 ★ [수건 정보 1순위]
잘못된 상식을 깨고 올바른 해결책을 제시.

```
[훅] "아직도 이렇게 하세요?" / "이거 하면 안 됩니다"

[오해/실수] 사람들이 믿는 잘못된 정보 1가지 언급

[반박] 왜 틀렸는지 — 구체적 수치/원리로 설명

[해결책] 대신 무엇을 해야 하는지 올바른 방법

[마무리] "혹시 이런 경험 있으신가요?" 댓글 유도
```

테리크 예시 주제: 섬유유연제가 수건을 망친다 / 수건 매일 세탁 안 해도 됩니다

---

## 구조 2. 교육팁/노하우형 ★ [수건 정보 2순위]
실질적인 정보를 번호 구조로 전달. 저장하고 싶은 느낌.

```
[훅] "N가지만 알면 됩니다" / "이것만 지키면 확실히 달라져요"

[팁 1] 구체적 방법 + 수치
[팁 2] 구체적 방법 + 수치
[팁 3] 구체적 방법 + 수치

[마무리] 한 줄 결론 + 댓글 유도
```

테리크 예시 주제: 수건 빨리 말리는 법 / 호텔 수건처럼 관리하는 법

---

## 구조 3. 커머스 스토리텔링형 ★ [답례품 1순위]
경험담처럼 시작해 자연스럽게 제품으로 연결. 광고 느낌 없이 구매 전환 유도.

```
[훅/썰] "얼마 전에 [경험/사건]이 있었는데..." (광고 느낌 X)

[공감] 시청자도 겪는 문제로 확장 — "다들 이런 고민 있잖아요"

[발견] 자연스럽게 해결책 등장 — "근데 [제품/방법]을 써보니까..."

[소구점] 핵심 장점 1~2개만. 구체적 변화/효과

[마무리] 강매 아닌 권유 — "궁금하신 분 댓글 남겨주세요"
```

핵심: 처음~중간까지 광고인 줄 모르게. "판매"가 아니라 "경험 공유" 느낌.
테리크 예시: 결혼 답례품 준비 / 돌잔치 답례품 수건 추천

---

## 구조 4. 상품 세일즈형 ★ [답례품 2순위]
고민을 짚고 테리크가 해결책임을 어필.

```
[훅] 핵심 불편/고민 언급

[해결책] 해결 방법이 있음을 알림

[가치] 구체적 장점 + 수치 (GSM, 인증 등)

[홍보] 테리크가 그 해결책임을 자연스럽게 연결

[마무리] 부드러운 권유 — "~해보세요"
```

---

## 구조 5. 감성/공감형
오늘 날씨·계절로 시작해 공감을 통해 자연스럽게 연결.

```
[훅] 오늘 날짜/계절/날씨로 즉시 공감 유도

[공감] 독자의 일상 속 상황 묘사 (1~2줄)

[연결] 수건/제품과 자연스럽게 이어지는 포인트

[감성 마무리] 따뜻한 한 줄 + 댓글 유도 or 시그니처 표현
```

---

## 구조 6. Q&A / 댓글 답변형
팔로워 질문에 전문가로서 답변. 소통감 + 권위 동시에.

```
[훅] "이 질문이 가장 많았어요" / "댓글에서 이런 질문이 왔어요"

[질문 소개] 실제 있을 법한 질문 텍스트 노출

[답변] 전문가적 관점에서 명쾌한 답변 + 수치

[보너스 팁] "덧붙이자면..." 추가 정보

[마무리] "궁금한 거 더 물어봐요" 댓글 유도
```

---

## 구조 7. 비교/반전형
Before vs After, 오해 vs 진실 구조.

```
[훅] "[이거] 말고, [이거] 쓰세요" / "before vs after"

[비교 A] 잘못된 방법 or 이전 상태

[비교 B] 올바른 방법 or 이후 상태 — 구체적 수치로 차이 설명

[결론] 한 줄 정리

[마무리] 댓글 유도
```
